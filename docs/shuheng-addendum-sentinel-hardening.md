# 枢衡施工补充单 · 哨兵加固(人批 2026-07-05,并入第一日初始化)

背景:D1 给 Claude Code qbase 库 DDL 权后,"拆触发器→改数→装回"的窗口为哨兵日检的 24 小时——与 append-only 的零窗口承诺不符。三项加固,把"日检"升级为"防拆":

1. **DDL 审计(核心)**:Postgres 事件触发器(event trigger)捕获全库一切 DDL(DROP/ALTER/CREATE TRIGGER 等),记入 `ddl_audit` 表——该表属 root 侧账户,Claude Code 账户无任何权限。**拆装动作本身必然留痕,藏不住**;哨兵日报首行即当日 DDL 清单,非空即人工复核。
2. **数据完整性双查**:哨兵每日记录各 append-only 表行数,任何一日行数下降=红色告警;对已冻结字段(pap_json、判断内容列)按日算校验和,变更即告警。
3. **哨兵自检**:哨兵 cron 自身与 ddl_audit 的权限配置纳入检查项(防"先废哨兵再动手");哨兵日报同时落地本机与直发人邮箱(双通道,删本地无用)。

验收:Claude Code 账户实测执行 DROP TRIGGER 后,ddl_audit 留痕且哨兵告警触达人——**用一次真实的"拆"来验收"防拆"**,验完恢复。
