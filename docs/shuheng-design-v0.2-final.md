## 1. 定位

枢衡是终局 4+1 的物理载体:L1(qbase)与第二引擎(淘沙)先住进来,观象/观澜/雷达'审计按各自触发日陆续迁入。它的存在意义按优先级排:**①给淘沙供干净的 PIT 数据(当下),②成为全体系唯一数据交换点(中期),③承载 monorepo 合仓(终局)。** 枢衡不做判断、不做信号加工、不做组合——那些是住户的事,它是楼。

## 2. 拓扑与执行通道

### 2.1 双云分工(定案)

- **阿里云机(4c16G/ESSD 200G,与老 DC 同地域)**:枢衡本体——qbase 库、淘沙运行时、采集、台账、每日备份。
- **AWS 机(2c8G+swap)**:工地与出口——Claude Code、git 仓、LLM API、结果副本、域名与未来服务。

三铁则:写权限只在阿里云 / 代码单向下行 / 服务读结果副本,库不开公网(只对 AWS 固定 IP 或隧道开只读)。

### 2.2 阿里云执行通道(D1 修订版,人批)

Claude Code(AWS)经受限系统用户接手阿里云日常运维:可 ssh、可部署、可跑批、可在 qbase 库执行 migration/建表——**不再需要人逐次触发部署**。但三样东西结构性不给,保住"焊死不靠自觉"的最后一寸:

1. **root/sudo 与备份通道不给**:备份 cron 跑在 root 下,OSS/S3 桶开版本保护——无论 Claude Code 犯什么错(误删表、拆触发器、错误 migration),恢复通道永远在它够不到的地方。
2. **老库账号管理不给**:只读 DSN 由老机管理,天然不可扩权。
3. **哨兵留在人手**:root 下挂 Claude Code 碰不到的每日自检 cron——检查台账/快照表触发器仍在、role_explore 权限未变、holdout 视图 WHERE 仍在、备份成功——结果直达人。**它接手一切,但"检查它有没有拆焊点"永远不由它自己做。**

初始化第一日配齐三样,约半小时。

### 2.3 三段时间制(设计决策 D2,代拟待批)

全部数据按三段划分,焊在视图层:
- **探索区**:< 2024-07-01(淘沙探索唯一可见区)
- **holdout 区**:2024-07-01 ~ 各假设动用日(人批,每假设一次)
- **实盘验证区**:系统上线后新增的实时数据(LLM 判断唯一合法验证区,"时间买无污染"的物理落点)
时间推移不改边界:holdout 静态、验证区随日历生长。

## 4. qbase 数据层设计

### 4.1 表族

| 族 | 表 | 要点 |
|---|---|---|
| 实体 | entity_master / entity_alias | ts_code 锚;别名表含巨潮 secCode/orgId 映射(对齐采集件) |
| 快照事实 | forecast_snap / holdertrade_snap | append-only,批次号+自打 observed_time,禁 upsert;触发器焊死 |
| 公告 | cninfo_ann(减持预披露起步) | announcementId 主键、announcementTime 毫秒、adjunctUrl;关键词/分类标签列 |
| 行情依赖 | trade_calendar / daily_price / adj_factor | 见 §9-核实1;收益计算口径=后复权收盘(D3) |
| 分类 | industry_class(申万一级,冻结)/ industry_custom(人的产业细分,活表) | 前者供 ADJ-BMP 行业内 ρ̄,口径冻结(改=新假设);后者随人演化,用于标注/分层探索/相关性提示,**永不参与已冻结假设的检验参数**(D5) |
| 治理 | llm_cutoff(五列)/ lineage 注释 / migration 日志 | |

### 4.2 视图族与权限隔离(设计决策 D4,代拟待批)

holdout 焊死不能只靠视图定义,还要防"读错视图":建三个 DB 角色——
- `role_explore`(淘沙引擎运行时):**只授** explore_* 视图(WHERE < holdout_start),连基表 SELECT 都没有——想越界读,结构上无权;
- `role_holdout`(平时无人持有):授全区间视图;人批准某假设动用 holdout 时临时启用,跑完即收;
- `role_service`(AWS 只读账号):只授结果类小表/视图,碰不到行情大表与快照基表。
探索代码"拿不到 holdout"由此从视图纪律升级为权限结构。

### 4.3 双时戳与时区

全库 UTC;valid_time 按数据语义(first_ann_date/ann_date/announcementTime),observed_time 一律自打;回填批次如实标注。

## 5. 淘沙接口(引用 spec v0.2 + 附录A)

reader 只连 role_explore;台账写入只发生在阿里云;result_json 记快照批次号;报告 markdown 随 sync 同步 AWS。切片与验收照 spec,不赘。

## 6. 安全与备份

- 秘钥:全部 .env,不进 git;AWS 机 .env 无任何写 DSN。
- 备份:每日 pg_dump(qbase+台账)→ 阿里云 OSS 桶归档 → 加密同步 AWS S3 异地;人的 ledger CSV 在覆盖内;**恢复演练第一日做,此后每季度一次**(到期台账里排上)。
- 老库访问:同云内网只读账号;坩埚/观澜 SQLite 经每日备份文件副本,不直连。

## 7. 运维

cron 三类:采集与快照(每日)/ 质量作业(快照 diff 每日、三层核对抽样每月出质量报告)/ 到期台账提醒(只提醒不建议)。告警走最简通道(邮件/钉钉任一),告警条目:采集失败、diff 发现回改、备份失败、磁盘水位 80%。不建监控大盘(红灯律)。

## 8. 演进路线(触发绑定,不排日期)

雷达判决期后:采集正式下沉合并、对账升 L3 入枢衡 → 坩埚判决日:数据经视图早已可读,收编=切换 → 观象验证周期后:采集下沉议题开启(总纲搁置区)→ monorepo 合仓完整化(第三年蓝图门后)。每步都是"资产迁入枢衡",枢衡自身不主动伸手。

## 9. v0.1 问题决议记录(人裁于 2026-07-05)

**◆ 行情覆盖度(原核实1)→ 降级为开跑前契约。** 行情/复权/日历属梯队 4("市面可重拉"),缺则回填,合宪。契约:**Q2 开跑前一天在老机实测三问**(全市场?含退市?复权因子+交易日历?),缺哪项即激活"日线行情回填子项"(tushare daily+adj_factor+trade_cal,含退市票)。不阻塞第一日与 Q1。

**◆ D1 执行通道**:修订版(§2.2,受限接手+三不给)。**◆ D2 三段时间制**:批。**◆ D3 收益口径=后复权收盘**:批。**◆ D4 三角色权限隔离**:批。**◆ D5 行业口径**:申万一级冻结 + industry_custom 活表并存(§4.1)。
