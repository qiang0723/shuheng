枢衡代码宪章(九条)——指导原则即日生效,v1.6 批复转正,届时入 CLAUDE.md

分层铁律

1. 依赖单向:compute(纯函数,零 IO 零 DB)← engine(编排+DB)← driver(入口)。compute 永不 import engine;违者验收拒收。
2. 口径唯一:任何统计口径/阈值/窗口只存在于 frozen_config 或 pap_json 一处;代码内魔法数=验收拒收;常量单一来源(引用不复制)。

规模与形态

3. 单文件 ≤500 行(现行红线转正);单函数 ≤60 行;超限即拆,拆分本身不需请示。
4. 纯函数优先:凡可无状态实现的计算做成纯函数,docstring 写明口径出处(附录/条款号)。

通用性(防平行代码,最重要)

5. 新假设接入的默认路径=复用:事件生成/路径模拟/检验/报告各层,新假设只允许新增"假设专属适配器"(该假设事件定义→标准事件流的转换),流水线主干不复制;主干确需改动走上报,不 fork。#3 检验接入为本条首检点。

可读与留痕

6. 每模块头部三行注释:职责一句话/口径依据(附录号)/验收档指针。存量模块不回补,新建与被触碰时补齐。
7. 自检随件:新模块必带可独立运行自检(现行惯例转正);回归用例只增不删。
8. 命名跟口径词汇表:代码标识符与文档术语一一对应(N_valid 不写 n_ok,armed 不写 pending),读代码与读附录看到同一个词。

演化

9. 重构=行为等价变更,允许且鼓励(audit 指纹+逐字节回归护航,谓词归位为先例);功能顺手加=禁止,"既然打开了就顺便"无例外。
