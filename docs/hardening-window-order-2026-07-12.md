可信度硬化窗口 · 施工单(终版,人批 2026-07-12;外部第二视角三轮审查全采纳)

窗口定义:自本单起至六项验收全过,暂停下一条假设的冻结与检验运行;#3 数据采集与验收、已登记假设资料准备照常。按序施工,验收凭实物:

① 台账状态机焊死(反向+正向双控)
修法:result 仅可随 running→done 首写;frozen_at 仅可随 registered→frozen 写;done_at 仅可随迁移写——字段变更绑定唯一合法迁移。
验收(a) 反向:taosha_app 权限逐条打非法路径(registered 态写 result / 提前写 frozen_at / 跳态迁移 / 二次写各字段 / 绕态改 done_at),全部被拒;
验收(b) 正向控制:registered 态 PAP 正常完善、registered→frozen→running→done 全程走通、registered/frozen→closed 走通、set_meta 类允许字段不被误伤、合法完成后 result 仍拒二次写。
两组用例固化入自检。

② StudySnapshot 快照锁定(fail-closed)
修法:研究启动时由受权角色(非 taosha_engine)生成一次性不可变 snapshot manifest(qbase 各源批次向量 + taosha 派生批次 pool_b1/pool_return);manifest 带不可变主键 + 规范化内容摘要 + created_at;引擎经按 manifest 路由的受限视图读取,不给 taosha_engine 扩任何底表权限;result.audit 同记 manifest ID 与 digest。
fail-closed 硬项:真实研究驱动无 manifest ID 直接拒绝运行,禁止静默回退 *_current。
验收:同一 manifest 双跑逐字节同;并发写入新批次时运行不受扰(实测);无 manifest 启动被拒(实测)。

③ 事件日 ST 修复 + 全量受控 diff(生成 post-ST 新基线)
修法:cleaning 的 ST 剔除改按事件日行 is_st 判定(现取 rows[0])。
验收:对 #4/#2b 全量重算清洗段,旧实现 vs 修复版做受控语义 diff(事件计数/剔除计数/板块分层归属逐项),影响以 diff 实测陈述,禁止以修复前计数预判影响;diff 产物即 post-ST 新基线(sha 登记)。旧闭卷 sha 永久保留,不被取代;已闭卷两案走 ⑤ 附注,不重跑不改判。

④ 共享存活样本主干(宪章第 5 条实质验收)
修法:提取"事件存活样本构造"单一主干(clean_event→sim_fit→coverage→robust 边界),事件版 runner 与策略版共同调用,平行链消除——不是拆短函数,是消灭第二条实现。
验收:重构后与 ③ 的 post-ST 新基线逐字节一致(#4 合成 / #2b 事件版 / #2b 策略版三份);验收标准=平行链消失(代码层单一调用点),非行数达标。#3 检验接入的前置。

⑤ experiment_addendum 审计附属表(不改终态载荷)
新建 append-only 表(Experiment 审计附属对象),字段:exp_id / 原 result sha256 / 问题类别 / 附注正文 / 是否影响 verdict / created_at / 审批来源指针;同款触发器焊死(禁 UPDATE/DELETE)。
首批两条附注:(a) exp_id=3 策略版定性="冻结口径下的不可执行诊断值,存在同刻成交前视与倾向乐观的偏置,不构成真实可交易表现证据"(不影响 verdict);(b) exp_id=3/5 ST rows[0] 缺陷登记,影响以 ③ diff 实测为准(不影响 verdict,理由随 diff 补录)。
原 result_json 一字不动。

⑥ 环境钉死 + 端到端集成回归
生产 Python 钉死到实际部署 minor 版本(3.11.x/3.12.x,以两台实况为准)并锁定依赖版本(锁文件);建端到端集成回归(manifest→读取→清洗→检验→报告一条龙),入常设自检。
新 PAP 模板硬门(随 v1.6 入档):自下一条包含策略版的假设起生效——策略版离场执行口径必须可执行(触发日次日开盘成交,或显式盘尾决策规则+滑点注记),同刻收盘判定+同刻成交不再允许;纯事件研究假设(如 #3)不受此约束。

六项完毕出硬化验收包(逐项实物 + 三份新基线 sha + addendum 表实物),交外部第二视角复审(复审标准:非法路径是否真走不通、同一快照是否真复跑出同一结果)+ 架构窗口终审,窗口关闭,恢复假设检验。

施工顺序纪律(补充令):
主序 ①②(可并行)→ ③ → ④ → ⑥集成回归,每项验收实物过架构窗口后进下一项;严禁 ④ 先于 ③(④ 的验收基线=③ 产物 post-ST 新基线);③ 的全量 diff 重跑必须在 ①② 已焊死的环境下进行(manifest 路由+台账新触发器生效),否则新基线自身不满足可复现要求。
⑤ 拆两半:建表+附注(a) 随时可做;附注(b) 待 ③ diff 出数后补录,⑤ 整体关闭在 ③ 后。⑥ 的 Python 钉版与锁文件即刻可做,端到端集成回归留末位(消费 ②④ 终态)。
每项完成 STATE 记档,验收逐项来,不攒总账。
