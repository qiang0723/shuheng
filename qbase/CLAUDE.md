# qbase · 交给 Claude Code 的工作宪法(CLAUDE.md)

> 动手前必须读完。施工范围以《新机初始化与qbase首批施工清单》为准(Q1–Q3;Q4 六对象 schema 另文待人批)。
> 有歧义停下来问,不要自作主张、不要填默认值。

## 0. 一句话

qbase = 全体系 L1 数据底座:管**公共事实(Fact)与归一视图**,是唯一的跨平台数据交换点。
它是地基,不是大脑:**只存事实、只做归一,零判断、零信号加工、零解读。**

## 1. 铁律

1. **只读上游,永不回写**:对老平台库(radar / research_view / marketdata)只用只读账号 SELECT;qbase 凭据不下发任何老机。坩埚/观澜数据经备份文件副本获取,不直连。
2. **双时戳强制**:一切落库带 valid_time(事件发生)+ observed_time(系统记录)。回填数据的 observed_time 如实标回填批次时刻并注明批次,**绝不冒充实时采集**;两戳绝不合并。UTC 入库,显示层再转。
3. **append-only**:回填与归一产物只增不删不覆盖;修数=新增修正行。触发器焊死,不靠自觉。
4. **含退市票**:全市场回填必须包含已退市证券(防幸存者偏差),这是 L1 的存在意义之一。
5. **holdout 线焊在视图**:explore_reader 族视图定义内置 `WHERE trade_date < holdout_start`(人批定 2024-07-01)。这条线不许出现在应用代码里"顺手放宽",改线=人走 PAP 流程。
6. **lineage 必填**:每张视图/每批回填注明 源/接入或批次时间/口径(三字段即可,不许扩成血缘系统)。防止新老两台机器出现"两份真相"。
7. **忠实归一**:视图只做字段对齐、单位统一、实体归一(ts_code 锚),**不过滤"坏"数据、不打质量分、不做任何加工判断**——那是消费方的事。

## 2. 绝不做(故意不做)

- 不建信号加工、不算任何统计指标(那是 L2/淘沙的活)
- 不做全量血缘/数据质量平台/调度系统/消息队列/列式仓库(重型件一律禁)
- 不迁移老平台代码或躯壳(雷达采集下沉等各自触发日,现在只经视图读它的表)
- Entity Master 只做证券主体,不扩公司/人物/概念图谱
- 秘钥纪律:tushare token 与各 DSN 只住 .env,不入库、不写文件、不进 git
